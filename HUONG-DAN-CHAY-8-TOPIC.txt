HƯỚNG DẪN CHẠY VÀ QUAN SÁT 8 TOPIC - DATETIMECHECKER
=====================================================

Thư mục dự án:
C:\Github-Jel-ui\AllDateTimeChecker


I. CHUẨN BỊ TRƯỚC KHI CHẠY
---------------------------

1. Mở PowerShell hoặc Command Prompt tại thư mục dự án:

   cd "C:\Github-Jel-ui\AllDateTimeChecker"

2. Máy cần có Java JDK, Node.js và npm.

3. Nếu đây là lần đầu chạy các bài test Node.js/Playwright:

   npm install
   npx playwright install chromium

4. Không cần mở run.bat trước khi chạy Topic 1 đến Topic 8.
   Các script test cần thiết sẽ tự khởi động server DateTimeChecker.

5. Nếu muốn mở web DateTimeChecker để thử thủ công:

   .\run.bat

   Trình duyệt sẽ mở tại:
   http://localhost:4173


II. TOPIC 1 - UNIT TESTING
--------------------------

Mục đích:
Kiểm tra riêng từng hàm xử lý, không cần thao tác trên trình duyệt.
Topic này chạy Java Unit Test cho backend và Jest Unit Test cho JavaScript.

Cách chạy:

   & ".\Topic 1 - Unit Testing\run-tests.bat"

Cách xem nó đang chạy:
- Cửa sổ terminal hiện phần RUNNING JAVA UNIT TESTS.
- Sau đó terminal hiện phần RUNNING JAVASCRIPT UNIT TESTS.
- Từng testcase sẽ hiện PASS hoặc kết quả thành công.

Kết quả thành công cần thấy:

   ALL UNIT TESTS COMPLETED SUCCESSFULLY!

File chính:
- Topic 1 - Unit Testing\run-tests.bat: chạy toàn bộ Unit Test.
- src\test\java\com\datetimechecker\DateTimeValidationServiceTest.java:
  các testcase Java backend.
- Topic 1 - Unit Testing\javascript-unit-testing\date-helpers.test.js:
  các testcase Jest cho JavaScript.


III. TOPIC 2 - API TESTING
--------------------------

Mục đích:
Gửi HTTP request trực tiếp đến API POST /api/datetime/check và kiểm tra
status code, JSON trả về, kết quả hợp lệ/không hợp lệ và thời gian phản hồi.

Cách chạy:

   & ".\Topic 2 - API Testing\run-tests.bat"

Cách xem nó đang chạy:
- Script biên dịch Java và tự khởi động server.
- Playwright gửi request API tự động, không cần mở trình duyệt.
- PowerShell tiếp tục chạy 10 testcase API.
- Terminal hiện từng dòng API01, API02... với PASS hoặc FAIL.

Kết quả thành công cần thấy:

   ALL API TESTS COMPLETED SUCCESSFULLY!

Báo cáo:

   reports\api-testing-report.tsv

Mở file TSV bằng Notepad hoặc Excel để xem input, expected result,
actual result, status code và elapsed time của từng testcase.

Demo bằng Postman:
- Import hai file JSON trong:
  Topic 2 - API Testing\postman
- Chọn environment DateTimeChecker API Testing.
- Mở request và bấm Send để xem response JSON.

File chính:
- Topic 2 - API Testing\run-tests.bat: chạy toàn bộ API test.
- Topic 2 - API Testing\playwright-api\api.spec.js: testcase Playwright API.
- Topic 2 - API Testing\run-api-testing.ps1: testcase PowerShell và tạo report.
- Topic 2 - API Testing\api-test-cases.md: danh sách testcase API.
- Topic 2 - API Testing\demo-guide.md: hướng dẫn demo API.


IV. TOPIC 3 - WEB E2E TESTING
-----------------------------

Mục đích:
Giả lập hành vi người dùng trên web từ đầu đến cuối: nhập ngày, bấm nút,
xem kết quả, xóa dữ liệu và đổi giao diện sáng/tối.

Cách chạy Playwright E2E:

   & ".\Topic 3 - Web E2E Testing\run-tests.bat"

Cách xem Playwright đang chạy:
- Khi chạy run-tests.bat, Chromium sẽ hiện trên màn hình.
- Playwright tự điền dữ liệu, bấm nút, xóa dữ liệu và đổi giao diện.
- Các thao tác được làm chậm để có thể quan sát khi trình bày.
- Terminal hiện từng testcase E2E01 đến E2E05.
- Mỗi testcase sẽ hiện passed hoặc failed.
- Khi Topic 3 chạy bên trong CI/CD, trình duyệt sẽ chạy ẩn để hoàn thành nhanh.

Kết quả thành công cần thấy:

   ALL WEB E2E TESTS COMPLETED SUCCESSFULLY!

Cách demo trực quan bằng Selenium:

   & ".\Topic 3 - Web E2E Testing\selenium\run-selenium-demo.bat"

Cách xem Selenium đang chạy:
- Microsoft Edge mở lên.
- Selenium tự nhập dữ liệu, bấm nút và kiểm tra kết quả.
- Đây là cách phù hợp để trình bày trực tiếp cho giảng viên.
- Lần đầu chạy cần Internet để tải selenium-server.jar.

File chính:
- Topic 3 - Web E2E Testing\run-tests.bat: chạy Playwright E2E.
- Topic 3 - Web E2E Testing\playwright-e2e\e2e.spec.js: các testcase E2E.
- Topic 3 - Web E2E Testing\selenium\run-selenium-demo.bat:
  chạy Selenium demo có thể nhìn thấy.

---------------------------------

File chính:
- Topic 5 - Performance Testing\run-tests.bat: chạy performance test.
- Topic 5 - Performance Testing\k6\load-test.js: kịch bản k6.
- Topic 5 - Performance Testing\benchmark\autocannon-test.js:

XII. CÁC LỖI THƯỜNG GẶP
------------------------

1. Lỗi "npm is not recognized" hoặc "npx is not recognized":
   Cài Node.js LTS, đóng terminal cũ và mở terminal mới.

2. Playwright báo thiếu browser:

   npx playwright install chromium

3. Báo cổng 4173 đang được sử dụng:
   Đóng cửa sổ run.bat hoặc server DateTimeChecker đang chạy, sau đó thử lại.

4. Mobile chỉ chạy mock mode:
   Mở Android Emulator hoặc kết nối điện thoại, sau đó kiểm tra bằng adb devices.

5. Gemini API bị lỗi:
   Dùng ai-assistant-chat-offline-sample.bat để demo dự phòng.
